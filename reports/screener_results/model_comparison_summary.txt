Key Findings:
-------------
1. ✅ LightGBM and more data helped (RandomForest was even worse)
2. ✅ Shorter horizons are less noisy (1-day best R²)
3. ❌ Current technical features don't predict absolute returns well
4. ⚙️  Models have weak ranking ability (IC ~0.05) for 3-5 day horizons
5. ❌ All R² scores negative = model worse than predicting mean

ROOT CAUSE: Feature engineering issue
--------------------------------------
Current features (technical indicators only) are insufficient for predicting 
short-term returns. This is actually expected - markets are largely efficient 
at short timescales.

Recommendations:
----------------
1. ADD FUNDAMENTAL FEATURES:
   - Price-to-earnings ratios
   - Revenue/earnings growth rates
   - Analyst estimates and surprises
   - Insider trading signals
   - Short interest
   
2. ADD MARKET CONTEXT FEATURES:
   - Sector momentum
   - Market regime indicators (VIX, breadth)
   - Relative strength vs sector/market
   
3. ADD ALTERNATIVE DATA:
   - News sentiment
   - Social media mentions
   - Options flow
   
4. CONSIDER ENSEMBLE:
   - Combine multiple weak models
   - Stack predictions from different horizons
   
5. REALISTIC EXPECTATIONS:
   - R² > 0.01 would be good for daily returns
   - IC > 0.05 is usable for ranking
   - Focus on ranking (IC) not absolute prediction (R²)

NEXT STEPS:
-----------
Since you're using this for RANKING candidates (not absolute return prediction),
a positive Spearman IC is more important than R². Your 3-day model with IC=0.05
can still provide value for portfolio construction by ranking stocks.

Consider:
- Using this as a ranking tool (top 20% of predictions)
- Combining with other screeners (ensemble approach)
- Adding more fundamental features over time
- Backtesting the top-N strategy to measure real performance
